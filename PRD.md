# 世界杯装懂指南 - 产品需求文档 (PRD)

## 一、项目概述

### 1.1 产品名称
世界杯装懂指南

### 1.2 产品定位
面向"四年一度球迷"的世界杯主题微信小程序，帮助不太懂球但不想被看出来的用户，通过趣味内容、测试和工具快速融入球迷圈子，假装资深老球迷。

### 1.3 目标用户
- 世界杯期间跟风看球的"伪球迷"
- 想参与社交话题但缺乏足球知识的新手
- 希望快速获取谈资、融入球迷圈子的用户

### 1.4 核心目标
- 趣味介绍2026美加墨世界杯48强球队及球员
- 提供球迷版MBTI测试，增强社交分享属性
- 提供多种"装懂"工具，降低新手门槛
- 实时获取足坛热点、赛程、赛果、新闻，保持内容新鲜度

---

## 二、数据获取策略总览

> **核心原则**：所有动态变化的数据（热点、赛程、赛果、新闻、实时比分）必须通过API实时获取；所有静态知识内容（球队百科、球员资料、黑话词典、测试题目等）使用本地数据或云数据库。

| 数据类型 | 获取方式 | 说明 |
|---------|---------|------|
| 足坛热点/新闻 | **实时API** | 必须实时获取，不可用本地数据 |
| 赛程/赛果/积分榜 | **实时API** | 必须实时获取，不可用本地数据 |
| 实时比分 | **实时API** | 必须实时获取，不可用本地数据 |
| 球队百科（48强） | 本地数据 | 静态知识，写入本地JS文件 |
| 球员详细资料 | 本地数据 | 静态知识，写入本地JS文件 |
| MBTI测试题目 | 本地数据 | 静态内容，写入本地JS文件 |
| 工具箱内容（黑话/梗图/绰号等） | 本地数据 | 静态知识，写入本地JS文件 |
| 用户收藏/测试记录 | 本地存储 | wx.getStorageSync |

#### 功能描述
小程序入口页面，展示足坛热点、今日赛程、实时比分、最新新闻等模块。

#### 核心功能

- **实时比分**：获取进行中比赛的实时比分
- **最新新闻**：通过新闻API获取足坛最新动态
- **今日黑话**：展示5条足球黑话术语（本地数据）
- **推荐球员**：展示明星球员卡片（本地数据）

{

## 四、实时数据API规范

### 4.1 推荐API方案

#### 方案A：聚合数据世界杯API（推荐）
- **官网**：https://www.juhe.cn/docs/api/id/11991
- **覆盖内容**：赛程、积分榜、赛果
- **特点**：中文接口、数据权威、实时更新
- **费用**：需申请API Key，有免费额度

#### 方案B：Football-Data.org（免费备选）
- **官网**：https://www.football-data.org/
- **API地址**：`https://api.football-data.org/v4/`
- **认证方式**：Header `X-Auth-Token: {api_key}`
- **免费额度**：10 calls/minute
- **覆盖内容**：比赛、赛程、积分榜、球队、球员

#### 方案C：TheSportsDB（免费备选）
- **官网**：https://www.thesportsdb.com/
- **API地址**：`https://www.thesportsdb.com/api/v1/json/3/`
- **特点**：完全免费，无需Key
- **覆盖内容**：赛事、球队、球员、图片

### 4.2 API接口清单

| 功能 | API端点 | 数据源 |
|------|---------|--------|
| 获取今日赛程 | `GET /v4/matches` | Football-Data |
| 获取世界杯赛程 | `GET /v4/competitions/WC/matches` | Football-Data |
| 获取实时比分 | `GET /v4/matches?status=LIVE` | Football-Data |
| 获取积分榜 | `GET /v4/competitions/WC/standings` | Football-Data |
| 获取球队信息 | `GET /v4/teams/{id}` | Football-Data |
| 获取球员信息 | `GET /v4/persons/{id}` | Football-Data |
| 获取新闻/事件 | `GET /eventsseason.php?id=4328` | TheSportsDB |

### 4.3 实时数据缓存策略

| 数据类型 | 缓存键 | 缓存时长 | 说明 |
|---------|--------|---------|------|
| 热点数据 | hotTopicsCache | 5分钟 | 新闻变化快，缓存时间短 |
| 赛程数据 | scheduleCache | 15分钟 | 赛程相对稳定 |
| 实时比分 | liveScoreCache | 1分钟 | 比分变化快，高频刷新 |
| 积分榜 | standingsCache | 10分钟 | 赛后更新 |
| 收藏球员 | favPlayers | 永久 | 用户收藏的球员ID数组 |
| 测试结果 | testResult | 永久 | 最近一次的测试结果 |

### 4.4 请求失败兜底方案

```javascript

  url: apiUrl,
  success: (res) => { /* 正常处理 */ },
  fail: () => {
    wx.showToast({
      title: '数据加载失败，请稍后重试',
      icon: 'none'
    })
    // 显示上次缓存的数据（如果有）
    // 或显示占位提示："比赛数据实时更新中..."
  }
})
```

## 五、MBTI测试详细设计

### 5.1 计分规则

- **总题库**：40题（通用题15题 + 5类特殊角色专属题各5题）
- **每次测试**：系统从40题中**随机抽取15题**进行测试
- **总分范围**：0-45分（15题 x 每题最高3分）
- **百分比计算**：得分 / 满分（45分）
- **特殊角色判定（双维度）**：
  1. **维度一 - 得分率**：该类题目得分率达到 **>=80%**（即该类题拿到80%以上分数）
  2. **维度二 - 贡献占比**：该类题目得分占15题总分的比例，在所有类型中**最高**
  3. 同时满足维度一和维度二，即可触发对应特殊角色
  4. 特殊角色优先级高于基础分数角色
  5. 若多个类型同时满足，取**贡献占比最高**的类型（不再固定优先级）
- **题目类型分布**：
  - 通用题（影响基础分数角色）：15题
  - 战术题：5题
  - 数据题：5题
  - 颜值题：5题
  - 毒奶题：5题
  - 考古题：5题
  - **总计：40题**

**特殊角色触发条件详解（双维度判定）**：

| 角色 | 维度一：得分率 | 维度二：贡献占比 | 说明 |
|------|--------------|----------------|------|
| 战术鬼才 | 战术题得分率 >= 80% | 战术题得分占15题总分比例最高 | 需在战术维度表现突出且领先其他类型 |
| 数据狂魔 | 数据题得分率 >= 80% | 数据题得分占15题总分比例最高 | 需在数据维度表现突出且领先其他类型 |
| 颜粉鉴赏家 | 颜值题得分率 >= 80% | 颜值题得分占15题总分比例最高 | 需在颜值维度表现突出且领先其他类型 |
| 毒奶大师 | 毒奶题得分率 >= 80% | 毒奶题得分占15题总分比例最高 | 需在毒奶维度表现突出且领先其他类型 |
| 足球考古学家 | 考古题得分率 >= 80% | 考古题得分占15题总分比例最高 | 需在考古维度表现突出且领先其他类型 |

> **判定示例**：用户抽中3道战术题、2道数据题、2道颜值题、1道毒奶题、7道通用题。战术题得7分/9分（得分率78%，不满足）；数据题得6分/6分（得分率100%，满足维度一），数据题得分6分占总分30分=20%。若其他类型贡献占比均低于20%，则触发「数据狂魔」。
>
> **注意**：由于15题是随机抽取，特殊角色并非每次测试都能触发。若未触发任何特殊角色，则按基础分数百分比判定5个基础角色。

### 12.2 越位1分钟看懂 (type: offside)

**核心规则**：进攻球员在队友传球的瞬间，如果比球和倒数第二名防守球员更靠近对方球门，就处于越位位置。

**4种不越位情况**：

| 编号 | 情况 | 说明 | 装懂话术 |
|------|------|------|----------|
| 1 | 本方半场不越位 | 无论队友在哪里传球，只要接球时你在自己半场，就不越位 | "只要在本方半场接球，永远不会越位" |
| 2 | 回传球不越位 | 如果队友往你的方向回传（即球的运动方向是朝向你的后场），不越位 | "回传球不存在越位，这是基本常识" |
| 3 | 球比你先出去不越位 | 如果传球时球比你的位置更靠近对方底线，不越位 | "球比人先出去，这球不越位" |
| 4 | 角球/球门球/界外球不越位 | 这三种情况下不存在越位规则 | "角球、球门球、界外球没有越位一说" |

**装懂金句**："越位看的是传球瞬间，不是接球瞬间，VAR画线就是这个原理。"
